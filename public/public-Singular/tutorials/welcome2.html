Singular Welcome Tutorial

Getting started

In this tutorial, we will show how to use this web site effectively.

Every tutorial usually consists of a number of pages, with highlighted Singular code.

If you click or tap on the highlighted Singular code, the code will be sent to the server, and evaluated. The result will be displayed in the right hand window. For example

2+2;
LIB "poly.lib";
ring r=0,(a,b,c,d,e,f),lp;
ideal I=cyclic(6);
ideal GI=groebner(I);
GI;

If you are already familiar with Singular and know some commands, you can type them directly in the right hand window.

You are on the first page of this tutorial. At the top of the window, you see how many pages are in this tutorial. After you are finished with a page, click or tap on the right arrow key. You can also go back to previous pages of the tutorial by clicking on the left arrow button.

Right now, please continue to the next page of the tutorial.

Using the Input Window

When you write Singular functions or long loops, it is often more convenient to write your code in a separate window and execute the code after you wrote it. This allows you to easily modify previous code and move the cursor around freely.

Clicking or tapping the Input button at the top of the page, navigates you to the Input Window. This is where you may type in Singular code and have it evaluated. Moreover, this window keeps track of everything you type into Singular at the right hand side.

In that window, place your cursor on a line that you wish to evaluate, and either click the Evaluate button, or press Shift-Return (both keys at the same time). Alternatively, you may select several lines, or a part of a line, and have them evaluated.

To save your work: Click on the Save button at the top right of the window. A dialog will appear where you can save the text in the Input window, or the text in the Singular Output (or both). Alternatively, you can just select the text you wish, and copy it.

Singular sessions and the Reset button

When your browser starts, it starts a Singular process on our server. All of your input, either typed and evaluated, or run by pressing buttons, goes to this process. If you open a second window or tab in the same browser, you will get the same session.

You might wish to start fresh, or restart Singular, if a computation is going on too long. In this case, press the Reset button at the top of the page.

When you first connect to this web site, a cookie is generated, and that is used to connect to your Singular process on the server. If you quit your browser, and it is set to remember windows, then it remembers the cookie, and you can reconnect back to the same session. If not, then a new session is created. The server will remove the session after some length of time, currently several days or a week.

Sometimes the server must be restarted for maintenance or other reasons, and at those times, all sessions are terminated.

Advanced topics

Uploading files

Use the Upload File button to upload files to the Singular server that can then be loaded into your Singular session. Suppose that you have uploaded a file named foo.lib which contains Singular code. You may then load this file into Singular using either:

LIB "foo.lib";

Note: these lines will likely give errors now, as the file foo.lib probably doesn't exist on the server!

Running shell commands in your Singular session

It is possible to run shell commands from within Singular, although many operations are restricted. For example, to see what files you have in your home directory on the server, try:

system("sh", "ls -la");

Restrictions in place for the Singular server

The server is being shared with other users, therefore memory and CPU time are limited (generous, but limited). Other restrictions include: upload file size, number of processes that can run, and the amount of time that the session is saved on the server.
